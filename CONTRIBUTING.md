# Contributing to FX Predict

Thank you for your interest in contributing to FX Predict! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/fx_predict.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `pip install -r requirements.txt`
5. Install in development mode: `pip install -e .`

## Development Setup

### Requirements

- Python >= 3.8
- PyTorch >= 2.0.0
- See `requirements.txt` for full list

### Running Tests

```bash
python test_structure.py
```

## Code Style

- Follow PEP 8 style guidelines
- Use meaningful variable and function names
- Add docstrings to all public functions and classes
- Keep functions focused and modular

## Making Changes

1. Write clear, descriptive commit messages
2. Keep commits focused on a single change
3. Add tests for new features
4. Update documentation as needed
5. Ensure all tests pass before submitting

## Submitting Changes

1. Push your changes to your fork
2. Submit a pull request to the main repository
3. Describe your changes in detail in the PR description
4. Reference any related issues

## Areas for Contribution

- **Model Improvements**: Enhance the TSGCN architecture
- **Data Processing**: Add support for more data formats
- **Visualization**: Improve plotting and analysis tools
- **Documentation**: Improve examples and tutorials
- **Testing**: Add unit tests and integration tests
- **Performance**: Optimize training and inference speed

## Questions?

If you have questions, please open an issue for discussion.

Thank you for contributing!
